# Unified Error Handling System

Унифицированная система обработки ошибок для всех микросервисов проекта simple shop.

## Зачем нужен этот пакет? 🎯

1. **Централизованное управление ошибками** - Все ошибки в системе используют единый интерфейс `DomainError`
2. **Предсказуемость системы** - Ограниченный набор типов ошибок упрощает обработку и тестирование
3. **Безопасность** - Разделение ошибок на публичные (для пользователя) и внутренние (скрытые)
4. **Консистентность** - Одинаковая обработка ошибок во всех микросервисах

## Типы ошибок 📋

### Публичные ошибки (информируют пользователя):
- `NotFoundError` - Ресурс не найден
- `UniqueConstraintError` - Нарушение уникальности
- `InvalidParametersError` - Неверные параметры
- `UnauthorizedError` - Неавторизованный доступ
- `AccessDeniedError` - Отказ в доступе
- `AuthTokenError` - Ошибка токена

### Внутренние ошибки (скрыты от пользователя):
- `InternalServiceError` - Внутренняя ошибка сервиса

## Основные принципы ⚡

1. **Обязательное логирование** - Перед созданием любой доменной ошибки она логируется
2. **Маппинг внешних ошибок** - Ошибки из внешних библиотек преобразуются в доменные
3. **Логическая взаимосвязь** - Ошибка отражает реальную ситуацию (NotFound при отсутствии записи)

## Быстрый старт 🚀

```go
// Создание ошибки NotFound
return errors.NewNotFoundError("user", userID)

// Создание ошибки с логированием
logger.Errorf(ctx, "user not found - id: %d", userID)
return errors.NewNotFoundError("user", userID)

// Маппинг внешней ошибки
if err != nil {
    logger.Errorf(ctx, "database error: %v", err)
    return errors.NewInternalServiceError("failed to get user", err)
}
```

- `utils.go` - Утилиты для проверки типов ошибок
- Отдельные файлы для каждого типа ошибки
